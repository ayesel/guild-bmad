# GUILD competitive positioning (2026-07-03)

**Source:** two independent premium deep-research reports, reconciled via the `/guild-expedition` Reconcile seam. Full reconciliation + raw reports: `guild-output/guild-artifacts/research/guild-competitive-positioning/` (`reconciliation.md`, `reports/chatgpt.md`, `reports/claude.md`). This doc is the durable, citable summary.

## Verdict

GUILD has **no direct all-in-one competitor**. Its differentiation is at the **workflow-architecture** level — discipline-specialized personas + full research→handoff lifecycle + brownfield traceability + hard QA gate + design-system engineering + multi-model raids + deep-research expeditions — not at any single feature. Two independent researchers converged on this, and on the strategy GUILD had already committed to: **be the reasoning/method/orchestration brain; delegate rendering.**

## What to do (confirmed by both reports)

1. **Position as the design-discipline orchestration layer** — above generators, beside bridges. Compete on method, discipline depth, QA rigor, and traceability. Never on raw UI generation or design-to-code.
2. **Do not rebuild** design-to-code, prompt-to-UI, token pipelines, or the Figma canvas. Treat v0, Figma Make, Builder Fusion, Anima, Locofy as downstream handoff targets invoked by Healer/Tinker.
3. **Deepen BMAD integration** and market GUILD as the design guild that replaces BMAD's single UX persona — converts a competitor into a distribution channel.
4. **Double down on Figma MCP + Code Connect (Tinker)** — the converging design-system-aware standard, consumable from Claude Code.
5. **Lead with the hard QA gate (Sage GO/NO-GO) + brownfield traceability spine** — the governance/enterprise wedge; no documented comparable has a design GO/NO-GO gate.
6. **Make Raids + Expeditions headline differentiators, scoped to the design domain** — generic multi-model councils are prior art at the technique level (OpenRouter Fusion Router; MindStudio/Suprmind/Jenova), so claim the *design-discipline specialization*, not the technique.

## The one watch-item — Claude Design

The reports **diverged** here, and it's the divergence that matters: the strategic report flags **Claude Design** as the primary platform-level threat (Anthropic, launched Apr 17 2026; two-way Claude Code sync via `/design` + `/design-sync` from Jun 17 2026; reads a codebase to build a design system), while the other report barely mentions it. Because GUILD runs on Claude, this is the risk to monitor. GUILD's defense is the multi-persona method, QA gate, and lifecycle coverage Claude Design does not provide (this is the existing gate-harness / DesignSync relationship, per [[claude-design-product]]).

**Reassess trigger:** if Claude Design adds multi-persona pipelines, hard QA gates, or research orchestration, revisit GUILD's differentiation.

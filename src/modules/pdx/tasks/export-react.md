# Export to React

## Purpose
Transform a PDX design artifact (user flow, wireframe, state diagram, etc.)
into a functional React component that can be used as a prototype or
handed off to the development pipeline.

## Input
- The most recent PDX artifact from `_bmad-output/pdx-artifacts/`
- Or a specific artifact path provided by the user

## Pre-flight
1. Read the source artifact
2. Check for project design tokens in `_bmad-output/pdx-artifacts/design-tokens.json`
3. Check for project config output format preference (react/html/svg)
4. Ask user: "Interactive prototype or static representation?"

## Export Modes

### Mode A: Interactive Prototype
Generate a fully functional React component that:
- Implements the flow/states from the artifact
- Uses Tailwind CSS for styling (no external CSS files)
- Includes state management via useState/useReducer
- Has working navigation between steps/states
- Shows all documented states (empty, loading, error, success)
- Is accessible (ARIA, keyboard nav, focus management)
- Is a single self-contained .jsx file

### Mode B: Static Visualization
Generate a React component that renders the artifact visually:
- Mermaid diagram rendered as SVG
- Annotated with design decisions
- Printable/shareable format
- Suitable for stakeholder review

    // Accessible, keyboard-navigable implementation

## Quality Checks
- [ ] All states from source artifact are implemented
- [ ] Error states are reachable and recoverable
- [ ] Component is accessible (run through mental a11y checklist)
- [ ] No external dependencies beyond React and Tailwind
- [ ] Component has zero required props (works standalone)
- [ ] Code comments reference source artifact sections

## Output Location
Save to: `_bmad-output/pdx-artifacts/components/[artifact-name].jsx`

## Post-Export
1. Tell the user where the file was saved
2. Note any states or interactions that were simplified for the prototype
3. Suggest: "Want me to capture this to Figma? (requires Figma MCP)"
